# 🛠️✨ Как запустить проект😊📚

Это простое объяснение, как запустить наш проект 🧑‍💻, который умеет читать тексты 📜 и анализировать их 🧠. Следуйте шагам ниже! 👇🚀 Мы добавим ещё больше деталей, чтобы у вас точно всё получилось! 💪📚

---

## 1️⃣: Скачиваем и устанавливаем 🐍 **Anaconda** 🛠️

1. **⬇️ Скачиваем Anaconda**:
   - Идите на 🌐 сайт [Anaconda](https://www.anaconda.com/products/distribution) 🌍.
   - Нажмите на кнопку “💾 Download” и выберите свою систему 💻 (Windows 🪟, Mac 🍎 или Linux 🐧).
   - Дождитесь, пока всё скачает 🕒.

2. **⚙️ Устанавливаем Anaconda**:
   - Откройте 📂 скачанный файл.
   - Нажимайте **“Далее”** 👉, пока не установится ✅.
   - Убедитесь, что **“Add Anaconda to PATH”** выключено 🚫 (это важно!).

3. **🔍 Проверяем установку**:
   - Откройте Anaconda Prompt 💬 или терминал.
   - Введите:
     ```bash
     conda --version
     ```
   - Если всё правильно, увидите версию программы 🎉.

4. **😎 Советуем**: Установите Anaconda Navigator 🧭 — это удобная программа для управления библиотеками и средами.

---

## 2️⃣: Подготавливаем проект 🧰

1. **⬇️ Скачиваем проект**:
   - Введите команду:
     ```bash
     git clone <ссылка на проект>
     ```
   - Или скачайте 📦 ZIP-архив и распакуйте его 📁.

2. **⚙️ Делаем рабочую среду**:
   - Откройте 📂 папку с проектом в терминале 💻.
   - Создайте среду 🌱:
     ```bash
     conda create --name my_env python=3.9
     ```
   - Включите среду 🔄:
     ```bash
     conda activate my_env
     ```

3. **📦 Ставим нужные программы**:
   - :
     ```bash
      conda install nltk tqdm scikit-learn openpyxl flask
      pip install nltk tqdm scikit-learn openpyxl flask
     ```

4. **💡 Проверка**:
   - Убедитесь, что библиотеки установлены ✅:
     ```bash
     pip list
     ```
   - Если что-то не установилось, повторите команду **pip install** 🛠️.

---

## 3️⃣: Готовим программы для работы 💻

### 🛠️ Вариант 1: Если у вас PyCharm 🧩
1. Откройте PyCharm и выберите папку 📁 проекта.
2. Настройте 🛠️ Python:
   - Зайдите в **File ➞ Settings ➞ Project ➞ Python Interpreter** 🐍.
   - Добавьте свою среду `my_env` 🌱.
3. **🎉 Всё готово** — можете запускать скрипты!

### 🛠️ Вариант 2: Если у вас Visual Studio Code 🖌️
1. Установите программу [VS Code](https://code.visualstudio.com/) 🖥️.
2. Добавьте расширение Python 🐍 (Microsoft).
3. Включите Python в терминале 💬:
   ```bash
   Ctrl+Shift+P > Python: Select Interpreter
   ```
4. Выберите `my_env` 🌱.
5. **🔄 Готово!** Теперь можно запускать скрипты 📜.

---

## 4️⃣: 🚀 Запускаем проект 🏃

### 🔍 **Анализ текстов 📜**
1. Зайдите в папку `src/` 📂.
2. Включите среду 🌱:
   ```bash
   conda activate my_env
   ```
3. Запустите программу 🧑‍💻:
   ```bash
   python data_analysis.py
   ```
   📊 Результаты будут в папке `results/` 📁.

4. **👀 Посмотрите**: Откройте файл `results/data_analysis.xlsx` 📊 и изучите результаты!

### 🔗 **Запускаем REST API 🌐**
1. Зайдите в папку `src/` 📂 и запустите 🔄:
   ```bash
   python data_analysis_with_api.py
   ```
2. В Postman или подобной утилите обращайтесть по адрессу 🔗:
   ```
   http://127.0.0.1:5000
   ```
3. Отправьте текст 📜 для анализа через 🧑‍💻 или пример самого запроса:
   ```bash
   curl -X POST http://127.0.0.1:5000/analyze -H "Content-Type: application/json" -d '{"text": "Ваш текст"}'
   ```

🎉 Ответ выглядит так:
```json
{
  "analysis": {
    "num_chars": 24,
    "num_words": 5,
    "spam_ratio": 40.0,
    "keywords": [
      ["sample", 1],
      ["text", 1]
    ]
  }
}
```
4. **📦 Сохраните**: Можете отправлять разные тексты и сохранять результаты JSON 🗂️.

# Инструкция по запуску тестов

## 1. Установка зависимостей

Перед тем как запускать тесты, нужно установить все зависимости, указанные в проекте. Для этого следуйте этим шагам:

### 1.1 Создание виртуального окружения (опционально)

Если у вас еще нет виртуального окружения, создайте его для изоляции зависимостей:

- Для Windows:
  ```bash
  python -m venv venv
  ```

- Для macOS/Linux:
  ```bash
  python3 -m venv venv
  ```

### 1.2 Активировать виртуальное окружение

- Для Windows:
  ```bash
  venv\Scripts\activate
  ```

- Для macOS/Linux:
  ```bash
  source venv/bin/activate
  ```

### 1.3 Установка зависимостей

Убедитесь, что у вас есть файл `requirements.txt`, который содержит все нужные зависимости. Для установки зависимостей выполните команду:

```bash
pip install -r requirements.txt
```

Если у вас нет `requirements.txt`, установите все необходимые библиотеки вручную:

```bash
pip install nltk pandas openpyxl pytest
```

## 2. Запуск тестов

Для запуска тестов используются два подхода: один для тестов с использованием `unittest`, второй — для `pytest`. Выберите тот, который вам подходит.

### 2.1 Запуск тестов с использованием `unittest`

1. Убедитесь, что ваш код (например, файл `test_text_analysis.py`) находится в папке, предназначенной для тестов.
   
2. Для запуска тестов через `unittest`, выполните команду:

- Для Windows:
  ```bash
  python -m unittest discover -s src/test -p "*.py"
  ```

- Для macOS/Linux:
  ```bash
  python3 -m unittest discover -s src/test -p "*.py"
  ```

Это выполнит все тесты, которые находятся в каталоге `src/test`.

### 2.2 Запуск тестов с использованием `pytest`

Если вы предпочитаете использовать `pytest` (более гибкий и функциональный тестировщик), выполните следующие шаги:

1. Для запуска тестов с использованием `pytest`, выполните команду в директории вашего проекта:

  ```bash
  pytest src/test/test_text_analysis.py --maxfail=1 --disable-warnings -q
  ```

Этот запуск будет искать все тесты в файле `test_text_analysis.py` и остановится после первого неудачного теста (`--maxfail=1`).

Опция `--disable-warnings` отключает предупреждения, а `-q` делает вывод более компактным.

## 3. Проверка результатов

После выполнения тестов вы получите результат в консоли. Если все тесты прошли успешно, в конце будет вывод типа:

```
=========================== 4 passed in 2.12 seconds ===========================
```

Если один из тестов не прошел, вы увидите ошибку с подробным описанием, указывающим на источник проблемы.

Эти шаги помогут пользователю легко настроить и запустить тесты в любой среде.
